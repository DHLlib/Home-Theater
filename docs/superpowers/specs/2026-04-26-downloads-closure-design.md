# Phase 2：下载功能闭环（草案）

> 日期：2026-04-26
> 来源：用户实时讨论补充

---

## 目标

补齐 MVP 下载功能的执行闭环，实现真正的断点续传下载、进度可视化、暂停/继续、删除与错误处理。

---

## 核心功能

### 1. 下载执行循环

- 后台 `asyncio` 循环，从数据库取出 `status=queued` 的任务
- 使用 `httpx` 流式下载，`headers={"Range": f"bytes={downloaded_bytes}-"}`
- 使用 `aiofiles` 追加写入本地文件
- 每写入 64KB，更新数据库 `downloaded_bytes` 和 `status=downloading`

### 2. 暂停/继续

- 暂停：`status=paused`，循环在下一个 chunk 后退出
- 继续：`status=queued`，循环重新拾取
- 取消/删除：停止下载，进入删除流程

### 3. 前端进度可视化

- 下载列表页每 2s 轮询 `GET /api/downloads`
- 进度条：`downloaded_bytes / total_bytes * 100%`
- 状态标签：queued | downloading | paused | done | error
- 操作按钮：暂停/继续、删除、重试（视错误类型）

### 4. 错误分类与处理

| 错误类型 | 原因 | 前端表现 |
|---------|------|---------|
| `connection_error` | 超时、拒绝连接 | 显示「重试」按钮 |
| `site_unavailable` | 站点已禁用或不存在 | 不显示「重试」，显示「站点不可用，请前往设置检查」 |
| `file_removed` | 404 或资源已删除 | 不显示「重试」，显示「资源已失效」 |

- 下载失败时，后端尝试 `health.probe(site)` 快速探测
- 站点不存在或 `enabled=false` → `site_unavailable`

### 5. 删除任务与源文件清理

**删除流程**：
1. 用户点击「删除」→ 弹出确认对话框
2. 对话框内容：「确定删除此下载任务？」
3. 对话框包含复选框：「同时删除本地源文件」
4. 用户点击「确定」：
   - 若勾选了「删除源文件」：尝试删除 `file_path` 对应的物理文件
   - 若文件已被用户在资源管理器中手动删除：提示「源文件删除失败，请检查文件是否存在」，但仍删除任务记录
   - 若未勾选：仅删除数据库中的任务记录

**异常处理**：
- `FileNotFoundError`：友好提示后，继续删除任务记录
- `PermissionError`：提示「无权限删除源文件，请检查文件权限」，仍删除任务记录
- 删除任务记录后，停止对应的下载循环（如果正在下载）

---

## 边界与异常

- 恢复下载时站点不可用 → `site_unavailable` 错误，不重试
- 用户手动删除源文件后再删任务 → 提示友好，任务仍删除
- 下载目录磁盘满 → `error_type=disk_full`，提示用户清理空间
